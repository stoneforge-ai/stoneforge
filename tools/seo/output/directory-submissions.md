## dev.to Cross-Post Draft

````markdown
---
title: "Introducing Stoneforge: Open-Source Orchestration for AI Coding Agents"
published: true
tags: ai, opensource, devtools, coding
canonical_url: https://stoneforge.ai/blog/introducing-stoneforge
---

If you've tried running multiple AI coding agents simultaneously, you've probably hit the coordination wall. Each agent works in its own terminal, unaware of the others. Merge conflicts pile up. Agents duplicate work. Context degrades silently. The "coordination tax" — the human overhead of managing multiple agents — grows with every additional agent you add.

**Stoneforge** is an open-source platform that solves this by orchestrating AI coding agents in parallel.

## How it works

**Director-Worker Architecture:** A Director agent decomposes goals into prioritized, dependency-ordered tasks. A background dispatch daemon assigns unblocked tasks to idle workers every five seconds, enabling parallel execution without human coordination overhead.

**Git Worktree Isolation:** Instead of containers or remote instances, Stoneforge uses git worktrees — git-level isolation without OS-level overhead. Multiple workers share build caches and dependencies while working on isolated branches.

**Structured Handoffs:** When workers approach context limits, they perform structured handoffs — committing code, documenting progress, and exiting cleanly. Fresh workers resume with complete task history, preventing silent context degradation.

**Automatic Merge Review:** Merge stewards handle code review, conflict resolution, test execution, and main branch pushes without human intervention.

**Persistent Knowledge:** Everything is event-sourced to JSONL (human-readable, git-mergeable) with SQLite caching. A Documentation Directory gives agents indexed access to specs and decisions before work begins.

## Multi-provider support

Stoneforge works with Claude Code, Codex, and OpenCode. No vendor lock-in.
````
